Node 地下铁沙龙 #9

Let’s Go Deep!

Cristian Palmer

这是一次硬核的地下铁沙龙，

我们深入 Node.js 运行时底层，

来讨论如何进行运行时的优化和诊断，

让它可以在 Serverless，IoT 等等场景释放更大的价值。


五位重量级的嘉宾，

有 Node.js 技术委员会（TSC）成员，

有来自浏览器厂商的骨灰级技术专家，

还有阿里、Rokid 的大牛，


欢迎你和我们一起，进入深海。

观看视频

凌 恒 @淘宝

未正 · 阳向幽

Node.js 在工具，Web 应用领域中已经相对成熟，随着新概念的层出不穷，大家都在尝试探索更多新的场景。在淘宝，我们将 Node.js 作为前端投身 FaaS 浪潮的首要选择。过程中，我们看到了机遇，也感受到了现状的尴尬。到底 Node.js 能否在 FaaS 场景中产生价值，是我们探索的目标。

PPT

林作健 @UC

UC 浏览器对 V8 引擎的各种优化

UC 浏览器高级技术专家。UC 工作近 9 年的一线搬砖。主攻 JS 引擎和浏览器的疑难、性能和内存问题。UC 的 JS 引擎之所以被器重，是因为自身超越 Chromium 的可用性、更好的性能和更低的内存消耗。作健来分享一下这些年在引擎优化方面做了什么工作。

PPT

普 冬 @阿里云

深入浅出 Alinode

Alinode 核心开发成员，在 Node.js 的 Runtime 方面有深入的研究。分享关于 Alinode 当前的一些功能及实现以及未来一段时间关于堆外内存监控的方案。

PPT

吴成忠 @Rokid

Optimize Node.js Performance with fork(2)

网名吞吞/@legendecas，来自 Rokid，Node.js Core 贡献者，嵌入式 JavaScript 引擎 JerryScript 贡献者，N-API 贡献者，目前作为 YodaOS 应用框架主要贡献者，本次分享主要是在 ShadowNode 与 Node.js 上使用 fork(2) 优化运行时性能的实践。

PPT

Joyee Cheung @Node.js TSC

Node.js Core 的启动

Node.js 技术委员会（TSC）成员，Igalia Compilers 团队成员。分享关于 Node.js Core 的启动流程相关的话题以及正在进行的 V8 Snapshot 集成工作。
